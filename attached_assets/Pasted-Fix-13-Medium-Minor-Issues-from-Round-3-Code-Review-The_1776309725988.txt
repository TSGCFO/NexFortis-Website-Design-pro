### Fix M1: multer missing fileFilter — all files fully buffered

**File:** `artifacts/api-server/src/routes/qb-portal.ts`, around line 106

**Current code:**
```js
const upload = multer({
  storage: multer.memoryStorage(),
  limits: { fileSize: MAX_UPLOAD_BYTES },
});
```

There is no `fileFilter` on the multer config. Any file type is accepted and fully buffered into memory before the handler runs. This wastes memory on obviously invalid files (e.g., someone uploading a .exe).

**Fix:** Add a `fileFilter` that checks the file extension against a whitelist of allowed extensions. The product catalog defines `accepted_file_types` per product, but since multer runs before the handler knows which product was selected, use a union of all possible valid extensions: `.qbm`, `.qbw`, `.qbb`, `.csv`, `.xlsx`, `.pdf`, `.zip`. Reject anything else with a multer error.

```js
const ALLOWED_EXTENSIONS = [".qbm", ".qbw", ".qbb", ".csv", ".xlsx", ".pdf", ".zip"];

const upload = multer({
  storage: multer.memoryStorage(),
  limits: { fileSize: MAX_UPLOAD_BYTES },
  fileFilter: (_req, file, cb) => {
    const ext = path.extname(file.originalname).toLowerCase();
    if (ALLOWED_EXTENSIONS.includes(ext)) {
      cb(null, true);
    } else {
      cb(new Error(`File type ${ext} is not allowed. Accepted: ${ALLOWED_EXTENSIONS.join(", ")}`));
    }
  },
});
```

### Fix M4: CORS preflight returns 200 without proper headers

**File:** `artifacts/api-server/src/app.ts`, around line 53

**Current code:**
The `cors()` middleware config doesn't specify `optionsSuccessStatus`. The default behavior varies by browser, and some older browsers expect 204 for preflight.

**Fix:** Add `optionsSuccessStatus: 204` to the cors config:

```js
app.use(
  cors({
    credentials: true,
    origin: (origin, callback) => {
      if (!origin || allowedOrigins.includes(origin)) {
        callback(null, true);
      } else {
        callback(null, false);
      }
    },
    methods: ["GET", "POST", "PUT", "PATCH", "DELETE", "OPTIONS"],
    allowedHeaders: ["Content-Type", "Authorization", "X-Upload-Token"],
    optionsSuccessStatus: 204,
  }),
);
```

Also add the Replit app URL to the allowedOrigins array so CORS works in production:
```js
const allowedOrigins = [
  "https://nexfortis.com",
  "https://www.nexfortis.com",
  "https://qb.nexfortis.com",
  "https://nex-fortis-website-design-pro.replit.app",
  ...(process.env.NODE_ENV !== "production" ? ["http://localhost:5173"] : []),
];
```

The schema defines `productId` as `integer`, but the request body sends `product_id` as a string (e.g., `"qb-pro"`). **Fix:** Parse `product_id` as an integer. Add validation:

```js
  res.status(400).json({ error: "Email and a valid numeric product_id are required" });
  return;
}
```

Then use `parsedProductId` instead of `product_id` in the query and insert:
```js
const existing = await db.select().from(qbWaitlistSignups)
  .where(and(eq(qbWaitlistSignups.email, email), eq(qbWaitlistSignups.productId, parsedProductId)))
  .limit(1);
...
await db.insert(qbWaitlistSignups).values({
  email,
  productId: parsedProductId,
  productName: product_name || `Product #${parsedProductId}`,
});
```

### Fix m6: Invisible disabled button (no background color)

**Files:** `artifacts/qb-portal/src/pages/admin/mfa-challenge.tsx` (line 160) and `mfa-enroll.tsx` (line 203)

**Current code:**
```tsx
style={{ backgroundColor: code.length === 6 && !verifying ? "#B76E79" : undefined }}
```

When the button is disabled (code < 6 digits or verifying), `backgroundColor` is `undefined`. Combined with `text-white` in the className, the button text becomes invisible against the white card background. The `disabled:opacity-50` class applies 50% opacity to an already invisible button.

**Fix:** Always set a background color. Use a muted version for the disabled state:
```tsx
style={{ backgroundColor: code.length === 6 && !verifying ? "#B76E79" : "#d4a0a8" }}
```

Or better, move the color to Tailwind classes:
```tsx
className="mt-4 w-full py-3 px-4 rounded-lg font-semibold text-white transition-all bg-[#B76E79] disabled:bg-[#B76E79]/50 disabled:cursor-not-allowed"
```

And remove the `style` prop entirely. Apply this fix in both `mfa-challenge.tsx` and `mfa-enroll.tsx`.
