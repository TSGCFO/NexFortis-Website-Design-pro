## Step 5: Wire Welcome Email into User Profile Creation

New users are created via a Supabase Auth trigger (`handle_new_user()` function) that automatically inserts a `qb_users` row when someone signs up. The trigger runs in Supabase's PostgreSQL — it cannot call the application's email service directly.

Instead, wire the welcome email into the first successful `GET /api/qb/me` call after the user is created. This is the profile fetch that happens immediately after login/registration in the auth flow.

### 5a. Approach — "First Login" Detection

In `artifacts/api-server/src/routes/qb-portal.ts`, find the `GET /me` route handler (search for `router.get("/me"` or `"/me"`). This route is called by the frontend auth context after every successful authentication to fetch the user's profile.

Add logic to detect first-time logins and send the welcome email:

1. After fetching the user profile from `qb_users`, check if this is the user's first login. Use a simple heuristic: if the `qb_users.createdAt` timestamp is within the last 60 seconds of `new Date()`, this is a brand-new account and the welcome email should be sent.

2. To prevent duplicate sends on rapid sequential `/me` calls (which happen during the auth flow), use an in-memory `Set<string>` to track user IDs that have already received the welcome email in this server process lifetime:
   ```
   const welcomeEmailSent = new Set<string>();
   ```
   Before sending, check `if (welcomeEmailSent.has(userId)) return;`. After sending, add `welcomeEmailSent.add(userId);`.

3. Import `welcomeRegistrationEmail` from `email-templates.ts` at the top of `qb-portal.ts`.

4. Inside the `GET /me` handler, after the profile is fetched and returned, add the first-login check and email send (fire-and-forget):
   ```
   // First-login welcome email
   const createdAt = new Date(user.createdAt);
   const now = new Date();
   const isNewAccount = (now.getTime() - createdAt.getTime()) < 60_000;
   if (isNewAccount && !welcomeEmailSent.has(userId)) {
     welcomeEmailSent.add(userId);
     const portalUrl = getValidOrigin(req.headers.origin as string | undefined) + "/qb-portal";
     const emailContent = welcomeRegistrationEmail(
       user.name || "there",
       user.email,
       portalUrl,
     );
     sendEmail({
       to: user.email,
       subject: emailContent.subject,
       html: emailContent.html,
       replyTo: "support@nexfortis.com",
     }).catch(err => console.error("[Email] Failed to send welcome email:", err));
   }
   ```

> **⚠️ DO NOT:**
> - Add a new column to the database to track "welcome email sent" — the in-memory Set is sufficient for preventing duplicates within a server process, and the 60-second window prevents sends on subsequent logins. If the server restarts within 60 seconds of a registration, the worst case is a duplicate welcome email, which is acceptable.
> - Hook into the Supabase trigger — the trigger runs in PostgreSQL and cannot call HTTP endpoints or the Resend API
> - Send the welcome email on every `/me` call — only on first login (within 60 seconds of account creation)
> - Send welcome emails to operators — check that `user.role === "customer"` before sending

### 5b. Add Role Check

The welcome email should only go to customer accounts, not to the operator. Before the `isNewAccount` check, add: `if (user.role !== "customer") return;` (or wrap the entire welcome email block in `if (user.role === "customer" && isNewAccount && ...)`).
