## TASKS

### Task 1 — Operator Runbook → `docs/operator_runbook.md`

Complete internal runbook for Hassan covering:

**Core Conversion Workflow (with time estimates per step):**

1. Order email arrives from Stripe (customer name, email, file link, services)
2. Download .QBM → `C:\Projects\qb-orders\<order-id>\input.qbm`
3. Dry-run: `qb-convert "input.qbm" --dry-run`
4. If add-ons purchased, process in order: Health Check (`--report`), Audit Trail Removal, List Reduction, Multi-Currency Removal, Super Condense, CRA Period Copy (all manual QB procedures)
5. Full conversion: `qb-convert "input.qbm" --output "converted.qbm" --report "report.html" --verbose`
6. Verify report.html → Email customer converted.qbm + report.html
7. Rush orders: complete within 15 min. Archive order folder.

Target: <30 min conversion-only, <60 min with add-ons.

**Troubleshooting Table:**

| Issue | Resolution |
|-------|-----------|
| Already Premier | Refund, email template #4 |
| Pro edition | Refund, explain Enterprise-only |
| US edition | Warn in email, proceed |
| Corrupted file | Offer Recovery ($199) or refund |
| Customer sent .QBB/.QBW | Reply with QBM guide, no refund |
| Conversion fails | Full refund, escalate |
| File > 500 MB | Contact customer for alt delivery |
