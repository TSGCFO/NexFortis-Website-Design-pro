## Step 1: Enhance the SEO Component

### 1a. Add Twitter Card Tags

Add `twitter:card` (value: `summary_large_image`), `twitter:title`, `twitter:description`, and `twitter:image` tags to the existing Helmet output. These should derive from the same props as the OG tags.

### 1b. Add JSON-LD Injection

Add an optional `jsonLd` prop (type: `object | object[]`) to the SEO component. When provided, render it as a `<script type="application/ld+json">` tag in the Helmet output. ### 1c. Add Article and Breadcrumb Support

Add optional props for `articlePublishedTime`, `articleModifiedTime`, and `articleAuthor` to support future educational/article pages. These emit `article:published_time`, `article:modified_time`, and `article:author` meta tags when the `og:type` is `article`.

> **⚠️ DO NOT:**
> - Replace the existing SEO component — extend it while keeping backward compatibility
> - Hardcode any JSON-LD in the SEO component — it receives JSON-LD as a prop from each page

## Step 4: Landing Page Data Structure

### 4a. Create `landingPages.ts` Content Config

Create a typed content configuration file at `artifacts/qb-portal/src/data/landingPages.ts` that contains all landing page data. Each page entry must include:

- `slug`: string — the URL path segment (e.g., `enterprise-to-premier-conversion`)
- `metaTitle`: string — 50-60 characters, contains primary keyword
- `metaDescription`: string — 150-160 characters
- `h1`: string — primary keyword-containing heading (one per page)
- `productSlug`: string — links to the product catalog for price display (if applicable)
- `ctaLabel`: string — action-oriented CTA text (e.g., "Order Now — From $75 CAD")
- `ctaHref`: string — link to the service order page (e.g., `/services/enterprise-to-premier-conversion`)
- `heroImageAlt`: string — descriptive alt text for the hero image
- `faqs`: array of `{ question: string, answer: string }` — 4-6 per page, all unique
- `relatedSlugs`: array of 2-4 landing page slugs for the Related Services section
- `breadcrumbs`: array of `{ name: string, path: string }` — excluding Home (auto-prepended by layout)
- `category`: `service` | `problem` | `comparison` | `educational` — determines JSON-LD schema type
- `primaryKeyword`: string — for internal reference and schema generation

**Content for each page** is described in Steps 5a through 5m below. Do NOT use the `bodyContent` as a single markdown blob — break it into structured sections that the layout component can render with proper semantic HTML.

Define the page sections as a typed array of content blocks:
- `hero`: above-fold section with H1, intro paragraph (2-3 sentences), and CTA
- `overview`: main body content (3-4 paragraphs, 400-600 words)
- `features` or `benefits`: bullet list or card grid of key selling points
- `process`: numbered steps showing how the service works (3-5 steps)
- `pricing`: price display with strikethrough (reads from product catalog)
- `trustSignals`: security badges, guarantee text, PIPEDA compliance note
- `faq`: accordion section with 4-6 Q&As
- `relatedServices`: card links to related landing pages

### 4b. TypeScript Types

Define a `LandingPageData` interface that strictly types all the above fields. Export both the interface and the data array. Use the interface as the return type for a `getLandingPageBySlug(slug: string)` helper function.

> **⚠️ DO NOT:**
> - Hardcode prices in the content — use a placeholder token like `{launchPrice}` and `{basePrice}` that the rendering component resolves from the product catalog at render time
> - Write duplicate FAQ questions across pages — every FAQ must be unique
> - Write meta descriptions shorter than 150 or longer than 160 characters
> - Write meta titles shorter than 50 or longer than 60 characters

## Step 6: Landing Page Layout Component

### 6a. Create `LandingPageLayout` Component

Create `artifacts/qb-portal/src/components/landing-page-layout.tsx` — a reusable layout component that wraps all landing pages and renders the structured content sections.

The component must accept the `LandingPageData` interface as props and render:

1. **SEO head** — using the enhanced SEO component with meta title, description, canonical URL, OG tags, Twitter cards, and JSON-LD (Service schema for service pages, FAQPage schema for all pages, BreadcrumbList schema)
2. **Breadcrumb navigation** — visible breadcrumb trail matching the JSON-LD BreadcrumbList. Home → Services → [Page Name]. Styled as small, secondary text above the H1.
3. **Hero section** — H1, intro paragraph, above-fold CTA button. The CTA must be visible without scrolling on both desktop (1280px) and mobile (375px). CTA text includes the launch price from the product catalog (e.g., "Order Now — From $75 CAD").
4. **Main content** — the body content rendered with proper semantic HTML (H2 subheadings, paragraphs, lists). Use a two-column layout on desktop: main content (70%) and sidebar (30%). On mobile: single column, sidebar content moves below main content.
5. **Process steps** — numbered steps showing how the service works. Use a clean, visual step layout (number circles with connecting lines or similar).
6. **Trust signals block** — security badge ("256-bit encrypted upload"), PIPEDA compliance note, 100% data guarantee, process steps summary. This goes in the sidebar on desktop, below content on mobile.
7. **Pricing preview** — display the base price crossed out and the launch price prominently. Pull prices from the product catalog — never hardcode. Include "Launch pricing — limited time" label.
8. **FAQ accordion** — collapsible accordion with 4-6 Q&As. Each question is an H3. Must be keyboard-accessible and functional on touch devices (no hover-only interactions). Include FAQPage JSON-LD.
9. **Related services section** — card links to 2-4 related landing pages. Desktop: sidebar cards. Mobile: horizontal scrollable card strip below the FAQ.

### 6b. JSON-LD Generation

Create helper functions that generate the correct JSON-LD schemas:

- `generateServiceSchema(pageData, productData)` — for service landing pages. Must include `provider.address` (204 Hill Farm Rd, Nobleton ON L7B 0A1), `areaServed` (CA, US, GB, AU), `offers` with `priceCurrency: CAD` and the launch price from the catalog.
- `generateFAQSchema(faqs)` — for all landing pages. Maps the FAQ array to FAQPage structured data.
- `generateBreadcrumbSchema(breadcrumbs)` — for all landing pages. Prepends Home, uses full URLs.

> **⚠️ DO NOT:**
> - Use an image carousel or slider in the hero — use a single static hero image/illustration placeholder
> - Make the FAQ hover-only — it must work with click/tap for accessibility
> - Hardcode the company address in the schema generator — use the shared constant from Step 2
> - Render JSON-LD that doesn't match the visible page content — Google penalizes schema/content mismatch
> - Include testimonials with fake names or quotes — use clearly marked "Testimonial slot — to be populated" placeholders
